## Project Overview

MapleDevs is a Canadian game industry job board that aggregates jobs from Canadian game studios. The site is a static HTML website that reads job data from a Google Sheet, which is populated by an automated scraper.

**Key Architecture:**
- Static HTML website (no backend server)
- Google Sheets as the database (jobs_raw, jobs_review, jobs_live tabs)
- Automated job scraper (Greenhouse/Lever ATS APIs)
- SEO page generator for city/role/level filters
- Swarm intelligence system for content generation
- LinkedIn bot for social distribution

## Development Commands

```bash
# Install dependencies
npm install

# Scrape jobs and update Google Sheet
npm run scrape

# Build SEO pages (city, role, level landing pages)
npm run seo

# Generate blog draft (requires GOOGLE_API_KEY)
npm run blog:draft "Topic Name"

# Publish approved blog post
npm run blog:publish draft-filename.json

# Validate blog post before publishing
npm run blog:validate

# Post to LinkedIn (requires LINKEDIN_ACCESS_TOKEN)
npm run post

# Verify LinkedIn configuration
npm run verify
```

## Pipeline Flow

```
scraper -> jobs_raw -> jobs_review -> jobs_live -> website
```

**Three-tab review pipeline:**
1. `jobs_raw` - Every unique job the scraper finds
2. `jobs_review` - New jobs requiring approval (auto-triaged)
3. `jobs_live` - Approved jobs that appear on the website

The scraper auto-triages jobs using a scoring system. High-confidence official ATS jobs are auto-approved. Jobs that need manual review stay as `needs_review`.

**To publish a job:** Change `status` to `approved` in `jobs_review`. The next scraper run copies it to `jobs_live`.

## Environment Variables

Required for scraping:
- `GOOGLE_SHEET_ID` - Google Sheet ID
- `GOOGLE_SERVICE_ACCOUNT_EMAIL` - Service account email
- `GOOGLE_PRIVATE_KEY` - Service account private key (base64 encoded)

Required for AI features:
- `GOOGLE_API_KEY` - Google Generative AI API key (Gemini)

Required for LinkedIn:
- `LINKEDIN_ACCESS_TOKEN` - LinkedIn API access token
- `LINKEDIN_AUTHOR_URN` - LinkedIn author URN (urn:li:person:XXXXX)

Optional scraper configuration:
- `VALIDATE_APPLY_LINKS` - Validate job application links (default: true)
- `LINK_CHECK_CONCURRENCY` - Concurrent link checks (default: 6)
- `LINK_CHECK_TIMEOUT_MS` - Link check timeout (default: 8000)
- `JOB_MAX_AGE_DAYS` - Auto-expire jobs older than N days (default: 90)
- `AUTO_APPROVE_SAFE_JOBS` - Auto-approve high-confidence jobs (default: true)
- `AUTO_APPROVE_SCORE` - Auto-approval score threshold (default: 80)

## Adding a New Studio

Edit `scrape-jobs.js` and add an entry to the `STUDIOS` array:

```javascript
{
  name: "Studio Name",
  platform: "greenhouse", // or "lever"
  token: "token-from-url",
  city: "City, Province",
  locationFilter: "Canada" // or null for all jobs
}
```

To find the token, visit the studio's careers page and look for:
- Greenhouse: `boards.greenhouse.io/{TOKEN}` or `job-boards.greenhouse.io/{TOKEN}`
- Lever: `jobs.lever.co/{TOKEN}`

## SEO Page Generation

`build-seo.js` generates SEO-optimized landing pages for:
- Cities: vancouver, toronto, montreal, ottawa, quebec-city, edmonton, calgary, victoria, london, halifax, kitchener, burnaby
- Roles: programming, art, design, producer, qa, audio, ui-ux

Each page includes:
- SEO-optimized title and meta description
- Canonical URLs
- Open Graph and Twitter Card tags
- Structured data (JSON-LD)

## Swarm Intelligence System

The swarm system (`swarm/`) uses AI agents to generate content:

**Agents:**
- `scout.js` - Researches Canadian gaming industry news
- `reviewer.js` - Reviews and triages job postings
- `author.js` - Writes blog post drafts
- `optimizer.js` - Optimizes content for SEO
- `director.js` - Publishes content to the website

**State management:**
- `swarm/state.json` - Global swarm state
- `swarm/state.js` - Browser-compatible state file

Run swarm:
```bash
node swarm_manager.js
node swarm_manager.js --dry-run  # Test without writing files
```

## Blog/Editorial Pipeline

1. Generate draft: `npm run blog:draft "Topic"`
2. Review draft in `blog-drafts/{slug}.json`
3. Change `status` to `"approved"` in the draft
4. Publish: `npm run blog:publish {slug}.json`

**Draft JSON structure:**
```json
{
  "status": "approved",
  "slug": "url-friendly-slug",
  "title": "Compelling Headline",
  "seo_title": "SEO Optimized Title",
  "meta_description": "Meta description (max 160 chars)",
  "summary": "Internal summary",
  "html_body": "Clean HTML only (h2 and p tags)",
  "sources": [
    { "title": "Source Name", "url": "https://...", "used_for": "Reason" }
  ]
}
```

**Banned terms in content:** "Swarm", "AI-Powered", "Intelligence" (use "MapleDevs Editorial" instead)

## Important Notes

- Never delete old rows from `jobs_raw` - history helps the scraper identify new/expired jobs
- Featured jobs (feature=Yes) are not expired even if they disappear from the source
- The website only shows jobs with `status` of `approved`, `live`, or `active`
- Jobs with `link_status` of `expired`, `dead`, `missing_from_source`, or `inactive` are hidden
- Jobs older than 90 days are automatically marked as expired (configurable via `JOB_MAX_AGE_DAYS`)
- The scraper uses `job_id` to prevent duplicates (gh_studio_jobid, lv_studio_jobid, etc.)

## Data-Only Helper Fields

These fields are for the pipeline and should not be shown on the website:
- `job_id`, `source_url`, `last_verified_at`, `status`, `tags`
- `review_score`, `review_recommendation`, `review_reason`, `auto_reviewed_at`
- `link_status`, `first_seen_at`, `last_seen_at`, `scraped_at`, `notes`, `date_reviewed`
